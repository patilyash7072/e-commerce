# 🛒 E-Commerce Application

A simple Java-based E-Commerce application built with Maven and structured logging support.

## 📌 Overview

This project is a core Java e-commerce application designed to demonstrate:

- Clean project structure

- Modern logging setup (SLF4J 2.x + Log4j2)

- Maven-based build management

- Java 25 configuration

- Console logging with color support (Jansi)

### It can serve as a foundation for building:

- Product management systems

- Shopping cart logic

- Order processing systems

- Console-based commerce simulations

- Backend business logic prototypes

## 🏗️ Tech Stack

- Language	Java 25

- Build Tool	Maven

- Logging API	SLF4J 2.x

- Logging Engine	Log4j2

- Console Colors	Jansi

## 📦 Project Configuration

Logging Dependencies

- slf4j-api (Logging API)

- log4j-api

- log4j-core

- log4j-slf4j2-impl (Correct SLF4J 2.x binding)

- jansi (Colored console output support)

## 🚀 Getting Started

### ✅ Prerequisites

- Java 25 installed

- Maven installed

- Git installed

🔹 Clone the Repository

```git clone https://github.com/patilyash7072/e-commerce.git```

```cd e-commerce```

### 🗄️ Generate Database Schema (PostgreSQL)

The following SQL script creates the database schema for the e-commerce system.

```sql
-- This script was generated by the ERD tool in pgAdmin 4.
-- Please log an issue at https://github.com/pgadmin-org/pgadmin4/issues/new/choose if you find any bugs, including reproduction steps.
BEGIN;


CREATE TABLE IF NOT EXISTS "e-commerce".tbl_customer
(
    id integer NOT NULL GENERATED ALWAYS AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
    first_name character varying(100) COLLATE pg_catalog."default" NOT NULL,
    last_name character varying(100) COLLATE pg_catalog."default" NOT NULL,
    address text COLLATE pg_catalog."default",
    phone character varying(20) COLLATE pg_catalog."default",
    CONSTRAINT tbl_customer_pkey PRIMARY KEY (id)
);

CREATE TABLE IF NOT EXISTS "e-commerce".tbl_inventory
(
    product_id integer NOT NULL,
    quantity_available integer NOT NULL,
    CONSTRAINT tbl_inventory_pkey PRIMARY KEY (product_id)
);

CREATE TABLE IF NOT EXISTS "e-commerce".tbl_order
(
    id integer NOT NULL GENERATED ALWAYS AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
    customer_id integer NOT NULL,
    product_id integer NOT NULL,
    quantity integer NOT NULL,
    status character varying(50) COLLATE pg_catalog."default" DEFAULT 'PENDING'::character varying,
    created_at timestamp without time zone DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT tbl_order_pkey PRIMARY KEY (id)
);

CREATE TABLE IF NOT EXISTS "e-commerce".tbl_payment
(
    id integer NOT NULL GENERATED ALWAYS AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
    order_id integer NOT NULL,
    payment_method character varying(50) COLLATE pg_catalog."default" NOT NULL,
    amount integer NOT NULL,
    status character varying(50) COLLATE pg_catalog."default" DEFAULT 'PENDING'::character varying,
    created_at timestamp without time zone DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT tbl_payment_pkey PRIMARY KEY (id)
);

CREATE TABLE IF NOT EXISTS "e-commerce".tbl_product
(
    id integer NOT NULL,
    name character varying(150) COLLATE pg_catalog."default" NOT NULL,
    company character varying(150) COLLATE pg_catalog."default",
    price integer NOT NULL,
    CONSTRAINT tbl_product_pkey PRIMARY KEY (id)
);

ALTER TABLE IF EXISTS "e-commerce".tbl_inventory
    ADD CONSTRAINT fk_inventory_product FOREIGN KEY (product_id)
    REFERENCES "e-commerce".tbl_product (id) MATCH SIMPLE
    ON UPDATE NO ACTION
    ON DELETE CASCADE;
CREATE INDEX IF NOT EXISTS tbl_inventory_pkey
    ON "e-commerce".tbl_inventory(product_id);


ALTER TABLE IF EXISTS "e-commerce".tbl_order
    ADD CONSTRAINT fk_order_customer FOREIGN KEY (customer_id)
    REFERENCES "e-commerce".tbl_customer (id) MATCH SIMPLE
    ON UPDATE NO ACTION
    ON DELETE CASCADE;


ALTER TABLE IF EXISTS "e-commerce".tbl_order
    ADD CONSTRAINT fk_order_product FOREIGN KEY (product_id)
    REFERENCES "e-commerce".tbl_product (id) MATCH SIMPLE
    ON UPDATE NO ACTION
    ON DELETE CASCADE;


ALTER TABLE IF EXISTS "e-commerce".tbl_payment
    ADD CONSTRAINT fk_payment_order FOREIGN KEY (order_id)
    REFERENCES "e-commerce".tbl_order (id) MATCH SIMPLE
    ON UPDATE NO ACTION
    ON DELETE CASCADE;

END;
```

### 🔹 Build the Project

```mvn clean install```

### 🔹 Run the Application

If you have a main class:

```mvn exec:java```

Or run directly from your IDE.
